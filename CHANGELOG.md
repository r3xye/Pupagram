- История сообщений сохраняется между перезапусками сервера
- Автоматическое создание базы данных при первом запуске

### 📊 Структура базы данных

**Таблица users:**
- id (PRIMARY KEY)
- username (UNIQUE)
- created_at (timestamp)

**Таблица messages:**
- id (PRIMARY KEY)
- sender (кто отправил)
- recipient (кому отправлено)
- text (текст сообщения)
- timestamp (время отправки HH:MM)
- created_at (полная дата и время)

### 🔄 Как работает память

1. **При отправке сообщения:**
   - Сообщение сохраняется в SQLite
   - Отправляется получателю через WebSocket (если онлайн)
   - Остается в базе навсегда

2. **При открытии чата:**
   - Клиент запрашивает историю с сервера
   - Сервер загружает последние 100 сообщений из базы
   - История отображается в интерфейсе

3. **При перезапуске сервера:**
   - Все сообщения остаются в базе данных
   - История доступна сразу после подключения

### 🔍 Просмотр базы данных

Вы можете просмотреть содержимое базы данных:

```bash
sqlite3 messenger.db

# Посмотреть всех пользователей
SELECT * FROM users;

# Посмотреть все сообщения
SELECT * FROM messages;

# Посмотреть переписку между двумя пользователями
SELECT sender, text, timestamp 
FROM messages 
WHERE (sender='Alice' AND recipient='Bob') 
   OR (sender='Bob' AND recipient='Alice')
ORDER BY id;

# Выход
.quit
```

### 💾 Резервное копирование

Для сохранения истории сообщений просто скопируйте файл базы данных:

```bash
cp messenger.db messenger_backup_$(date +%Y%m%d).db
```

### 🗑️ Очистка истории

Для удаления всей истории:

```bash
rm messenger.db
# База будет создана заново при следующем запуске сервера
```

Для удаления истории конкретного пользователя:

```bash
sqlite3 messenger.db "DELETE FROM messages WHERE sender='Username' OR recipient='Username';"
```
