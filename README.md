# Локальный мессенджер

Простой локальный мессенджер с веб-интерфейсом и WebSocket коммуникацией.

## Установка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

## Запуск

### 1. Запустите сервер

```bash
python3 server.py
```

Сервер запустится на `ws://localhost:8765`

### 2. Запустите клиенты

В отдельных терминалах запустите клиенты для разных пользователей:

**Клиент 1:**
```bash
python3 client.py user1
```

**Клиент 2:**
```bash
python3 client.py user2
```

**Клиент 3:**
```bash
python3 client.py user3
```

Каждый клиент автоматически откроет браузер с веб-интерфейсом.

## Режимы работы

### Веб-режим (по умолчанию)
```bash
python3 client.py <username> [ws://server:port] web
```

Открывает браузер с графическим интерфейсом.

### Консольный режим
```bash
python3 client.py <username> ws://localhost:8765 cli
```

Работает в терминале с командами:
- `/send <username> <message>` - отправить сообщение
- `/contacts` - показать контакты
- `/online` - показать онлайн пользователей
- `/history <username>` - показать историю
- `/quit` - выход

## Технологии

- **Backend:** Python 3.8+, WebSockets
- **Frontend:** HTML5, CSS3, Vanilla JavaScript
- **Протокол:** WebSocket (JSON)

## Примеры использования

### Локальная сеть
Запустите сервер на одной машине:
```bash
python3 server.py
```

Подключайтесь с других машин:
```bash
python3 client.py username ws://192.168.1.100:8765
```

### Несколько пользователей на одной машине
```bash
# Терминал 1
python3 server.py

# Терминал 2
python3 client.py alice

# Терминал 3
python3 client.py bob

# Терминал 4
python3 client.py charlie
```

## Протокол WebSocket

### Регистрация клиента
```json
{
  "type": "register",
  "username": "alice"
}
```

### Отправка сообщения
```json
{
  "type": "send_message",
  "to": "bob",
  "text": "Привет!"
}
```

### Получение сообщения
```json
{
  "type": "message",
  "from": "alice",
  "to": "bob",
  "text": "Привет!",
  "timestamp": "14:30"
}
```

### Статус пользователя
```json
{
  "type": "user_status",
  "username": "alice",
  "status": "online",
  "timestamp": "14:30"
}
```

## Ограничения

- История сообщений хранится в памяти (при перезапуске сервера теряется)
- Нет шифрования сообщений
- Нет аутентификации
- Нет поддержки файлов
